# Report AI

## Run

1. `npm install`
2. `npm run dev`

Netlify function routes are available under `/api/*`.

## Required env vars

- `NEON_DATABASE_URL`
- `HUB_JWT_PUBLIC_KEY_PEM`
- `HUB_JWT_ISSUER` (optional, default `hub.rotor-platform.com`)
- `MODULE_AUDIENCE` (set to `report-ai`)
- `ANTHROPIC_API_KEY` (optional for local fallback mode)
- `BROWSERLESS_TOKEN` (for PDF export)
- `ASSET_BLOB_BASE_URL` (optional, base URL for design/photo assets)
- `UNSPLASH_ACCESS_KEY` (for `/api/unsplash-direct` — placeholder photo resolver used by Claude in alpha-v3 freeform rendering)

## Module audience pattern

## Alpha-v3 flow helpers

- `/api/unsplash-direct?q=<query>&w=<w>&h=<h>` — 302-redirect to a keyword-matched Unsplash CDN URL sized to requested dimensions. Embedded directly in Claude-generated report HTML as `<img src="…">`. Content-addressed blob cache per `(q, w, h, orientation)` — same query returns the same photo every render for PDF reproducibility.
- `/api/v2-pdf?key=<blob_key>` — serves multi-page "preview all" PDFs generated by `render_freeform_thumbnails` alongside per-page PNG thumbnails. One URL the user can click to see every sample page.
- `/api/v2-asset?key=<blob_key>` — serves per-page PNG thumbnails + brand-logo / font assets.

## MCP handlers (alpha-v3)

Selected handlers in `netlify/functions/mcp-v2.js`:
- `render_freeform_thumbnails` — renders the full batch in ONE `/render/pdf` call + ONE `/render/rasterize` call, then splits per-page PNGs out. Returns `thumbnails[]` (url + optional png_base64 + optional preview_url for the multi-page PDF). DPI defaults to 72 when `return_base64=true`.
- `render_freeform_pdf` — payload-driven PDF render. Used by the hub's render-worker via `enqueueRender(..., payload)`.
- `persist_freeform_pages` — called by smyra-core `module_review` approve_all. Atomic DELETE + bulk INSERT into `v2_report_pages` + `v2_report_modules` + `v2_reports.document_css` so the editor + legacy `render_pdf` can find the pages.
- `save_blueprint` — accepts `{brand_id, name, visibility, design_system_css, sample_pages_html[], design_rules, doctype_hint}`. Background-renders `sample_pages_html[0]` as `cover_thumbnail_url` (migration 029).

Schema migrations through 029 (see `db/migrations/`):
- `028_blueprints_alpha_v3.sql` — `design_system_css` / `sample_pages_html` / `design_rules` columns + `visibility IN ('smyra', 'tenant', 'brand')` constraint
- `029_blueprint_cover_thumbnail.sql` — `cover_thumbnail_url TEXT` for the setup picker's inline cover gallery
